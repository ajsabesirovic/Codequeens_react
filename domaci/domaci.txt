DOMAĆI ZADATAK - AuthContext

================================================================================
ZADATAK: Kreirati AuthContext za upravljanje autentifikacijom korisnika
================================================================================

OPIS ZADATKA:
------------
Kreiraj AuthContext koji će upravljati autentifikacijom korisnika u aplikaciji.
Context treba da omogući prijavu (login), odjavu (logout), i proveru da li je
korisnik trenutno prijavljen.

ZAŠTO KORISTITI CONTEXT ZA AUTENTIFIKACIJU?
-------------------------------------------
- Informacije o korisniku su potrebne na više mesta u aplikaciji
- Ne želimo da prosleđujemo user podatke kroz props svuda
- Context omogućava centralizovano upravljanje autentifikacijom
- Sve komponente mogu pristupiti informacijama o korisniku direktno

ŠTA TREBA DA URADITE:
-------------------

1. KREIRAJ AuthContext.jsx fajl
   - Lokacija: src/context/AuthContext.jsx
   - Default vrednosti: { user: null, isAuthenticated: false, login: () => {}, logout: () => {} }

2. KREIRAJ AuthProvider komponentu
   - Koristi useState za čuvanje user podataka
   - Učitaj user iz localStorage pri inicijalizaciji (ako postoji)
   - Implementiraj login funkciju koja:
     * Prima email i password kao parametre
     * Proverava da li su email i password validni (za početak: email="admin@test.com", password="admin123")
     * Ako su validni, čuva user objekat u state i localStorage
     * User objekat treba da ima: { email, name: "Admin User" }
   - Implementiraj logout funkciju koja:
     * Briše user iz state-a
     * Briše user iz localStorage
   - Vrati Context.Provider sa value prop-om koji sadrži: { user, isAuthenticated, login, logout }

3. AŽURIRAJ Login.jsx komponentu
   - Dodaj formu sa poljima za email i password
   - Koristi useContext(AuthContext) da pristupiš login funkciji
   - Na submit forme, pozovi login funkciju sa email i password vrednostima
   - Prikaži poruku o grešci ako login ne uspe
   - Prikaži poruku o uspehu ako login uspe

4. AŽURIRAJ App.jsx
   - Importuj AuthProvider
   - Obavij (wrap) aplikaciju sa AuthProvider-om
   - AuthProvider treba da bude unutar ili oko ThemeProvider-a

5. AŽURIRAJ ProtectedRoute.jsx (OPCIONO)
   - Umesto provere localStorage.getItem("isAuthenticated")
   - Koristi useContext(AuthContext) da proveriš isAuthenticated
   - Ovo je bolje rešenje jer koristi Context umesto direktnog pristupa localStorage

6. DODAJ PRIKAZ KORISNIKA
   - U Header.jsx komponenti, dodaj prikaz email-a korisnika ako je prijavljen
   - Dodaj dugme za logout ako je korisnik prijavljen
   - Koristi useContext(AuthContext) da pristupiš user i logout podacima
